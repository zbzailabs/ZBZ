Uso o ChatGPT desde 2023. Passei de usuário comum a usuário pago, depois do Plus para o Pro. No segundo semestre de 2025, quando o Gemini ganhou força de repente, cheguei a mudar temporariamente para o Gemini. Com o lançamento do GPT-5.5, voltei para o ecossistema da OpenAI. Há duas razões principais. Primeiro, a capacidade do modelo GPT-5.5 ainda está muito à frente da de outros modelos. As disputas de pontuação entre grandes modelos não têm tanto significado para usuários comuns. O que usuários comuns querem é produtividade real. Hoje, o GPT realmente chegou a um nível de produtividade capaz de entregar resultados. Segundo, a OpenAI fez muito trabalho de produto. Codex, Pulse e deep research reduzem, cada um à sua maneira, a barreira de uso.

Atualmente, uso os produtos da OpenAI principalmente em dois tipos de trabalho: propostas de projeto e programação. O fluxo geral para fazer propostas de projeto é este: uso Feishu Minutes/Lark Minutes para transformar os áudios de visitas técnicas e comunicação com clientes em atas de reunião; envio as atas revisadas e outros materiais do projeto para o ChatGPT; uso deep research para pedir ao ChatGPT um primeiro rascunho da proposta; exporto uma versão Word para checagem e revisão; por fim, uso o Codex para ajustar a formatação do texto. Se também for necessário fazer um PowerPoint, continuo usando o plugin presentations do Codex para criar o PPT. É claro que o PPT feito pelo presentations não é perfeito, e alguns PowerPoint skills open source no GitHub podem ser usados para melhorá-lo.

Para corrigir bugs e ajustar funcionalidades, o Codex dá conta completamente. Se a ideia for desenvolver do zero um sistema grande e completo, o Codex ainda enfrenta muitos desafios, especialmente a tendência de sair do controle. Nesse momento, o mais importante é definir limites claros: stack técnica, escopo funcional e outros limites. Ao iniciar um projeto, primeiro confirme dois arquivos: `AGENTS.md` e `DESIGN.md`. `AGENTS.md` serve para limitar a stack técnica, as regras de engenharia e o limite de execução; `DESIGN.md` serve para limitar cores, fontes, estilo dos componentes e outras regras visuais. O segundo passo é escolher a stack técnica. Escolha tecnologias que a IA usa bem, como React + TypeScript, em vez de tecnologias que humanos podem dominar melhor, como Java.

A IA mudou o ritmo de trabalho, e a questão de saúde não pode ser ignorada. AI Agents realmente aumentam muito a eficiência. Se uma tarefa sua ainda não terminou, o Agent conclui rapidamente a parte dele e empurra você para avançar, fazendo com que você continue trabalhando sem parar. Além disso, é um trabalho intenso, sem espaço para distração. O cérebro continua funcionando, e isso se torna muito cansativo com o tempo. Por isso, você precisa de uma forma externa de se obrigar a parar. A melhor forma é a técnica Pomodoro: trabalhar 25 minutos, descansar 5 minutos e dar uma pausa ao cérebro. Naturalmente, se a coordenação com o Agent for boa, você também pode descansar enquanto o Agent trabalha.
