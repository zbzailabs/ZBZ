Demi kebebasan dan fleksibilitas, banyak kreator independen beralih ke blog yang di-host sendiri. Karena registrasi server domestik merepotkan, sebagian memilih menaruh infrastruktur di luar negeri agar bebas dari prosedur. Saat memilih server luar negeri, pertimbangan utama adalah biaya dan performa jaringan. Saya menguji empat kombinasi: server Vultr Singapura, Vultr Singapura + Google DNS, Google VM + Google DNS (node Hong Kong), serta Google VM + Google DNS + Google CDN (node Hong Kong).

Soal kecepatan, tumpukan Google VM + Google DNS + Google CDN di Hong Kong adalah yang tercepat, sedangkan Vultr Singapura tanpa tambahan adalah yang paling lambat.

Uji kecepatan Google VM + Google DNS + Google CDN (node Hong Kong)

Dilihat dari harga, Vultr memberikan kredit 50 dolar kepada pengguna baru selama 30 hari jika mengisi minimal 5 dolar, dan paket termurah di data center Singapura memang 5 dolar per bulan. Google Cloud memberikan kredit 300 dolar berlaku satu tahun. Untuk blog pribadi dengan trafik kecil, itu berarti satu tahun Compute Engine gratis secara praktis.

Kesimpulannya, cobalah hosting blog di Google Cloud dan pilih lokasi fisik yang paling dekat dengan pembaca. Jika audiens berada di Tiongkok daratan, node Hong Kong adalah pilihan terbaik.

Netlify juga patut dicoba. Platform ini menyediakan hosting statis berbasis JAMstack dan backend tanpa server secara gratis, jadi kamu bisa bereksperimen dengan kombinasi Ghost + Netlify + Gatsby untuk membangun front-end blog di atas Gatsby.js.
